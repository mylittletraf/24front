# UI/UX ТЗ — публичный фронтенд

Референс по дизайну и UX: **YouTube**. Стек — на усмотрение разработчика (рекомендован Next.js + Tailwind CSS).
API-контракт — `docs/FRONTEND_SPEC.md`.

## 1. Дизайн-система

### 1.1 Темы

Поддерживать обе темы (переключатель, дефолт — системная):

|---|---|---|

| `--text-secondary` | `#606060` | `#aaaaaa` |
| `--text-link` | `#065fd4` | `#3ea6ff` |

| `--accent-hover` | `#cc0000` | `#cc0000` |
| `--border` | `#e5e5e5` | `#3d3d3d` |
| `--skeleton` | `#e0e0e0` | `#2a2a2a` |

### 1.2 Типографика

Шрифт: **Inter** (Google Fonts) или системный стек (`-apple-system, BlinkMacSystemFont, "Segoe UI", …`).

| Роль | Размер | Вес |
|---|---|---|
| Заголовок страницы | 20–24 px | 700 |
| Заголовок секции | 16–18 px | 600 |
| Название видео / карточка | 14 px | 500 |
| Тело / описание | 14 px | 400 |
| Мета (просмотры, дата) | 12–13 px | 400 |
| Метка / чип | 12–13 px | 500 |

### 1.3 Брейкпоинты

| Имя | Диапазон |
|---|---|
| `mobile` | < 640 px |
| `tablet` | 640 – 1023 px |
| `desktop` | ≥ 1024 px |
| `wide` | ≥ 1440 px |

### 1.4 Базовые компоненты

**Кнопки:**
- `primary` — заливка `--accent`, белый текст, radius 20 px.
- `secondary` — обводка `--border`, текст `--text-primary`, radius 20 px.
- `ghost` — без фона и обводки, icon + label.
- `icon` — 36×36 px круг/квадрат с radius 50%.

**Чипы / теги:**
- Pill-форма (radius 16 px), высота 32 px, padding 0 12 px.
- Состояния: `default`, `active` (заливка `--accent`), `hover`.

**Инпут поиска:**
- Высота 40 px, radius 20 px, иконка лупы справа, `Clear`-кнопка при наличии текста.

**Skeleton-загрузка:**
- Серые блоки (`--skeleton`) с pulse-анимацией (opacity 0.4 → 1 → 0.4, 1.5 s). Форма повторяет итоговый элемент.

**Бесконечный скролл:**
- `IntersectionObserver` на sentinel-элемент внизу списка.
- Показывать ряд skeleton-карточек пока грузится следующая страница.
- При ошибке — кнопка «Загрузить ещё» вместо skeleton.

**Состояние «пусто»:**
- Иконка + заголовок «Ничего не найдено» + подпись.

## 2. Глобальные компоненты

### 2.1 Шапка — мобильная (< 1024 px)

```
┌─────────────────────────────────────┐
│  [Лого]              [🔍] [☰]      │
└─────────────────────────────────────┘
  [Видео]  [Актёры]  [Подборки]       ← горизонтальный скролл, под шапкой
```

- **Лого** — ссылка на `/`. SVG или текст, цвет `--accent`.
- **Иконка поиска** — открывает fullscreen-инпут с затемнением.
- **Гамбургер** — открывает drawer справа:
  - Шапка drawer: аватар + имя пользователя (если auth) или кнопки «Войти» / «Зарегистрироваться».
  - Ссылки: Профиль, Избранное, История просмотров, Закладки.
  - Разделитель.
  - Заголовок «Категории».
  - Список всех категорий в столбик с прокруткой (scroll внутри drawer).
  - Ссылка «Все категории» в конце.
- Нижняя навигационная панель (tab bar) — фиксированная снизу экрана:
  `[🏠 Главная] [🎬 Видео] [👤 Актёры] [📚 Подборки]`

### 2.2 Шапка — десктопная (≥ 1024 px)

```
┌──────────────────────────────────────────────────────────────────────────────┐
│ [Лого]  Видео  Актёры  Подборки          [🔍 ______________ ]  [🔖]  [👤 ▼] │
└──────────────────────────────────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────────────────────────────────┐
│  Кат1   Кат2   Кат3   Кат4   Кат5                                           │
│  Кат6   Кат7   Кат8   Кат9   Кат10                                          │
│  ...    ...    ...    ...    ...                                             │
│  (до 7 строк × 5 столбцов = 35 категорий, последний элемент — «Все →»)     │
└──────────────────────────────────────────────────────────────────────────────┘
```

- **Лого** — ссылка на `/`.
- **Навигационные ссылки** — `Видео` → `/videos/`, `Актёры` → `/actors/`, `Подборки` → `/collections/`.
  Активная ссылка подчёркнута (нижняя граница 2 px, цвет `--accent`).
- **Поиск** — инпут шириной 350–500 px (flex grow). При вводе показывать dropdown с
  `/search/suggestions/` (debounce 200 мс, 8 подсказок максимум). Enter → страница `/search/?q=`.
- **Закладки** ([🔖]) — ссылка на `/me/favorites/`.
- **Профиль** ([👤 ▼]) — если не авторизован: кнопки «Войти» / «Зарегистрироваться».
  Если авторизован: аватар-инициалы + dropdown (Профиль, Избранное, История, Выйти).

**Блок категорий** (sticky под шапкой, только десктоп):
- Сетка 5 × 7 (5 категорий в строке, до 7 строк = 35 + «Все категории» как последний элемент).
- Каждый элемент — категория: текст с иконкой-чипом (или `preview_image` 24×24 px круглое).
- Клик → `/category/{slug}/`.
- «Все категории» → `/categories/`.
- Данные: `GET /categories/` (до 35 по `sort_order`).
- Блок сворачивается кнопкой «▲ Скрыть» / «▼ Категории» справа от сетки.

### 2.3 Подвал (Footer)

Только десктоп, минималистичный:
- Логотип слева.
- Навигационные ссылки: Видео / Актёры / Подборки / Категории / Контакты.
- Текст копирайта по центру.
- Переключатель темы (☀ / 🌙) справа.

## 3. Карточка видео (VideoCard)

Используется повсеместно: главная, ленты, сайдбары, поиск.

### 3.1 Структура

```
┌───────────────────────────────────┐
│                                   │ ← poster / trailer (16:9, object-fit: cover)
│  [👁 1.2k]          [▶ 10:00]   │ ← оверлей: нижний левый = просмотры, нижний правый = длительность
│                                   │
└───────────────────────────────────┘
  Название видео в одну строку...   ← 14 px, 500, text-overflow: ellipsis
  2 дня назад                       ← 12 px, --text-secondary, relative дата
```

- **Соотношение сторон:** 16:9. `<div>` с `padding-bottom: 56.25%` + `position: relative` внутри.
- **poster** — `<img>` fill, `object-fit: cover`. Lazy load (`loading="lazy"`).
- **Длительность** (нижний правый угол): тёмный pill `rgba(0,0,0,.8)`, белый текст, размер 12 px.
  Формат: `MM:SS` или `H:MM:SS` если ≥ 1 часа.
- **Просмотры** (нижний левый угол): тёмный pill, иконка 👁 + число.
  Форматировать: `< 1000` → число; `≥ 1000` → `1.2k`; `≥ 1 000 000` → `1.2M`.
- **Название** — одна строка, `overflow: hidden; white-space: nowrap; text-overflow: ellipsis`.
- **Дата** — относительная (`2 часа назад`, `3 дня назад`, `5 мес. назад`).

### 3.2 Поведение трейлера

**Десктоп (hover):**
1. При наведении курсора: ≥ 300 мс задержки, затем `<video>` заменяет `<img>`.
2. `<video>` с `src=trailer`, `autoplay muted loop preload="none"`, `object-fit: cover`.
3. При уходе курсора: остановить и скрыть видео, вернуть постер.
4. Если `trailer === null` — поведение hover отсутствует (только постер).
5. Не начинать загрузку трейлера до hover.

**Мобильная (auto-play в viewport):**
1. `IntersectionObserver` (threshold 0.7): при попадании карточки в видимую область запустить трейлер.
2. При выходе из viewport — пауза/скрытие.
3. `<video muted playsinline loop preload="metadata">`.
4. Не более **2 трейлеров одновременно** (очередь).

### 3.3 Варианты размеров

| Контекст | Колонки | Мин. ширина карточки |
|---|---|---|
| Главная (desktop) | 4 | 220 px |
| Главная (tablet) | 3 | 180 px |
| Главная (mobile) | 1 | full-width |
| Сайдбар «популярные» | 1 (горизонтальная карточка) | — |
| Похожие видео (desktop) | 2–4 (flex) | 160 px |
| Похожие видео (mobile) | 1 | full-width |

**Горизонтальная карточка** (для сайдбара):
```
[постер 16:9 ~120px] │ Название в 2 строки
                       2.1k просмотров · 3 дня назад
```

### 6.3 Плеер

- Использовать **Video.js** (с hls плагином). Источник: `sources.hls`.
- Poster-превью плеера: `poster` из detail-ответа.
- Контролы: play/pause, таймкод, прогресс-бар (scrubbing), громкость, полный экран, качество (HLS ABR).
- При `sources.hls === null` — показать сообщение «Видео недоступно».
- **VAST pre-roll:** до старта воспроизведения — `GET /videos/{uuid}/vast/?placement=pre_roll`;
  при `204` — сразу воспроизводить основное видео.
- **Трекинг просмотра:** `POST /videos/{uuid}/view/` при начале воспроизведения (один раз).
- **Прогресс:** `POST /videos/{uuid}/progress/` каждые 15 с + при паузе + при закрытии страницы
  (`beforeunload`). Тело: `{watch_progress: position/duration, last_position_seconds: Math.floor(position)}`.
- При reload/переходе — восстанавливать `last_position_seconds` из `/me/continue-watching/` (если auth).

### 6.4 Реакции и действия

- **Лайк/Дизлайк** (auth):
  - `POST /videos/{uuid}/reaction/` `{"reaction":"like"|"dislike"}`.
  - `DELETE /videos/{uuid}/reaction/` для снятия.
  - Оптимистичное обновление счётчика на ±1.
  - Неавторизованному — предложить войти (modal).
- **Анонимный лайк** — кнопка «Нравится» для анонима отправляет `POST /videos/{uuid}/guest-like/`.
  После — кнопка заблокирована (LocalStorage дедуп на клиенте на 7 дней дополнительно).
- **Сохранить в избранное** — иконка закладки, `POST/DELETE /videos/{uuid}/favorite/`.
- **Пожаловаться** — иконка флага. Modal: выбор темы из `/report-topics/` + textarea описания.

### 6.5 Категории и Актёры

- **Категории** — chips-ссылки на `/category/{slug}/`.
- **Актёры** — chips с `photo`-аватаром (24 px круг) + имя, ссылка на `/actor/{slug}/`.

### 6.6 Описание

- По умолчанию — 3 строки, кнопка «Ещё» / «Скрыть».
- Если `description === null` — блок не отображать.

### 6.7 Теги

- Chips с `#` префиксом, ссылки на `/tag/{slug}/`.
- Горизонтально с переносом (`flex-wrap: wrap`).

### 6.8 Комментарии

- Заголовок: «Комментарии (N)» где N = `comments_count` из detail.
- Сортировки: «Лучшие» / «Новые» / «Старые» — tabs или dropdown.
- Загрузка: `GET /videos/{video_uuid}/comments/?sort=top` cursor-пагинация, кнопка «Загрузить ещё».
- **Элемент комментария:**
  ```
   Имя · 2 дня назад
   Текст комментария
   [👍 5] [👎 2]  [🗑 Удалить — только автор]
  ```
  `author === null` — показывать «Аноним» серым.
- **Форма добавления** (только auth): textarea + кнопка «Отправить».
  После отправки показать toast: «Комментарий отправлен на модерацию».
  Неавторизованному — заглушка «Войдите, чтобы оставить комментарий».

### 6.9 Сайдбар «Популярные видео»

- 5 горизонтальных карточек: `GET /videos/popular/?page_size=5`.
- Заголовок «Популярное».
- Только desktop (`≥ 1024 px`).

### 6.10 Похожие видео

- `GET /videos/{slug}/related/` — до 12 карточек без пагинации.
- Desktop: 2–4 колонки (зависит от оставшейся ширины после сайдбара).
- Mobile: 1 колонка.
- Заголовок «Похожие видео».

## 8. Страница актёров (`/actors/`)

**API:** `GET /actors/?sort=popular&page_size=30` + `GET /tags/` (для атрибутов-фильтров)

### 8.1 Блок фильтров

Горизонтальная полоса фильтров под шапкой. На мобильном — кнопка «Фильтры ▼» открывает sheet снизу.

| Фильтр | UI | Условие |
|---|---|---|
| Рост | Ползунок диапазона (Range slider) | от 140 до 200 см |
| Вес | Ползунок диапазона | от 40 до 120 кг |
| Пол | Dropdown: Любой / Женщины / Мужчины | — |
| Страна | Dropdown с поиском (список стран-тегов) | — |
| Телосложение | Dropdown | — |
| Размер бюста | Dropdown | Активен только при Пол=Женщины; при другом поле — disabled |
| Тип груди | Dropdown | То же |
| Цвет волос | Dropdown | — |
| Цвет глаз | Dropdown | — |

- Значения для Dropdown-атрибутов получать из `GET /tags/?page_size=200` фильтруя по флагам
  (`is_country`, `is_body_type`, `is_bra_size`, `is_boobs_type`, `is_hair_color`, `is_eye_color`).
- Параметры запроса актёров: `?gender=woman&country=<slug>&sort=popular`.
- Рост/вес — на бэке нет фильтра по диапазону; фильтровать на клиенте при небольших выборках
  или показывать как декоративный UI до реализации серверного фильтра.
- Кнопка «Сбросить» справа от панели фильтров.
- При изменении фильтра — `GET /actors/?gender=…&country=…&sort=popular`.

Варианты сортировки (tabs или dropdown рядом с фильтрами): Популярные / По алфавиту / Новые.

### 8.2 Сетка актёров

- **5 колонок** desktop (4 tablet, 3 mobile).
- Бесконечный скролл. Skeleton — 10 карточек.
- Карточка: `ActorCard` (см. §4).

```
│                │  Дата рождения: 01.01.1990                   │
│                │  ─────────────────────────────────           │
│                │  Биография (свернуть/развернуть 3 строки)    │
└────────────────┴──────────────────────────────────────────────┘
```

- `cover_image` — фоновое изображение hero-блока с затемнением (`--bg-overlay`), `object-fit: cover`.
- `photo` — карточка поверх фона, `border-radius: 8px`, тень.
- Атрибуты — только непустые (`!== null`). Не выводить строки с `null`.
- Псевдонимы — если `aliases.length > 0`.
- Пол явно не выводить (implicit по контексту).

Mobile: `photo` на всю ширину (3:4), данные — блок под фото.

### 9.2 Список видео

- Заголовок: `Видео с [Имя] (N)` где N = `videos_count` из объекта актёра.
- `GET /actors/{slug}/videos/?lang=ru` — cursor-пагинация, бесконечный скролл.
- Grid: 4 колонки (3 tablet, 2 mobile).

## 10. Страница видео по категории (`/category/{slug}/videos/`)

**API (SSR):** `GET /categories/{slug}/?lang=ru` + `GET /seo/category/{slug}/?lang=ru`
**API (CSR):** `GET /categories/{slug}/videos/?lang=ru` + `GET /categories/{slug}/related/?lang=ru`

### 10.1 Шапка страницы

```
[preview_image 64 px круглое]  Название категории
                               N видео
```

Описание категории (если есть), свернуто по умолчанию.

### 10.2 Блок тегов / фильтров («Похожие теги»)

- Горизонтальная прокручиваемая полоска тегов из `GET /categories/{slug}/related/`.
- Каждый тег — чип с `+` для добавления в фильтр или `-` для исключения.
- Логика:
  - Клик на чип → добавляется к `include_tags` в URL (`?include_tags=slug1,slug2`).
  - Повторный клик → переключается в `exclude_tags` (чип красный с `−`).
  - Ещё раз → снимается.
- URL-параметры синхронизируются с фильтром каталога видео (`GET /videos/?categories=slug&include_tags=…`).
- Заголовок блока «Уточнить выборку».
- Отображать: `intersection_count ≥ 2`, сортировка по `intersection_count ↓`.

### 10.3 Сетка видео

- Та же сетка, что на главной (4→3→2→1 колонок).
- Бесконечный скролл, cursor-пагинация.
- Пустое состояние: «Видео по этой категории не найдены».

### 12.1 Hero-блок (богатое оформление)

Desktop:

## 13. Поиск (`/search/`)

**API:** `GET /search/?q=…&lang=ru` + `/search/{videos|tags|categories|actors}/`

### 13.1 Страница

- URL: `/search/?q=query`.
- Tabs: «Все» / «Видео» / «Теги» / «Категории» / «Актёры».
- «Все» — смешанная выдача: секции по типам, `≤ 5` элементов каждой.
- Остальные табы — соответствующий `/search/{type}/` (PageNumber, 20–30 элементов).
- Пустой результат — «По запросу «query» ничего не найдено».

### 13.2 Автокомплит (dropdown в шапке)

- `GET /search/suggestions/?q=…` при вводе (debounce 200 мс, мин. 2 символа).
- Dropdown под инпутом, max 8 подсказок.
- Иконка слева соответствует типу: 🏷 тег, 📁 категория, 👤 актёр.
- Клик на подсказку → `/tag/{slug}/videos/`, `/category/{slug}/videos/`, или `/actor/{slug}/`.

## 14. Аутентификация

### 14.1 Модальные окна (не отдельные страницы)

- **Вход** — modal или drawer: поля username/password, кнопка «Войти», ссылка «Нет аккаунта? Зарегистрироваться».
- **Регистрация** — modal: username, password, email (опционально), display_name (опционально).
  После успеха: автологин, показать `quick_login_token` один раз (copyable input).
- **Быстрый вход** (quick_login_token) — `/auth/quick-login/`. Ссылка формата `/quick-login?token=…`
  обрабатывается на фронте (не требует отдельной страницы).

### 14.2 Личный кабинет (`/me/`)

Tabs: «Избранное» / «Понравилось» / «История» / «Продолжить просмотр» / «Мои жалобы».
Каждая вкладка — PageNumber-список видео-карточек.

**Настройки профиля** — отдельный раздел/tab «Профиль»:
- Отображаемое имя (`display_name`) — inline-редактирование, `PATCH /me/`.
- **Ссылка быстрого входа** — блок «Быстрый вход»:
  - Описание: «Ссылка для входа без пароля — используйте на доверенном устройстве».
  - Кнопка «Получить ссылку» → `POST /me/quick-login-link/` → показать modal с результатом.
  - После нажатия кнопки: ссылка `https://{domain}/quick-login?token={quick_login_token}`
    появляется прямо в блоке настроек — `<input readonly>` + кнопка «Копировать» (Clipboard API + toast «Скопировано»).
  - Ссылка остаётся видимой до тех пор, пока пользователь на странице. После обновления страницы поле пустое — нужно нажать снова.
  - Повторное нажатие «Получить ссылку» (или кнопка «Обновить») → `POST /me/quick-login-link/` → старый токен инвалидируется, новый отображается в том же поле.
  - Предупреждение под полем: «Не передавайте ссылку третьим лицам».
- Кнопка «Удалить аккаунт» (деструктивная, красный `secondary`-стиль) → confirm dialog →
  `DELETE /me/` → разлогин и редирект на главную.

## 15. Общие UX-паттерны

### 15.1 Toast-уведомления

- Позиция: нижний правый угол, стек до 3 уведомлений.
- Автоскрытие через 4 с.
- Типы: `success` (зелёный), `error` (красный), `info` (синий).
- Использовать при: лайк, добавление в избранное, удаление из истории, отправка комментария, жалоба.

### 15.2 Обработка ошибок

| Ошибка | Действие |
|---|---|
| `401` | Открыть modal входа |
| `403` | Toast «Нет доступа» |
| `404` → редирект | `GET /redirects/?slug=…`; если `redirect:true` → `router.replace`; иначе → страница 404 |
| `429` | Toast «Слишком много запросов, попробуйте через … сек» (из `Retry-After`) |
| Сетевая ошибка | Toast «Ошибка сети. Проверьте соединение» |

### 15.3 Языковой переключатель

- Присутствует на страницах с переводами (видео, теги, актёры, коллекции).
- Отображает только языки, присутствующие в `slugs`.
- Переключение: `router.replace(/path/{slugs[newLang]}/?lang={newLang})`.
- Бейдж «Контент на другом языке» при `fallback_language !== null`.

### 15.4 Мета и SEO

- **SSR-обязательно** для всех публичных страниц (видео, тег, актёр, коллекция, главная).
- `<head>` из `/seo/{type}/{slug}/`:
  - `<title>`, `<meta name="description">`, `<link rel="canonical">`.
  - `<link rel="alternate" hreflang="…">` для каждого языка в `alternates`.
  - OpenGraph и Twitter meta.
  - `<script type="application/ld+json">` = `json_ld`.
  - `<meta name="robots" content="…">` точно как в ответе (не переопределять).

### 15.5 Производительность

- Lazy load всех изображений (`loading="lazy"`, `decoding="async"`).
- `priority` только для LCP-изображения (первый экран).
- Шрифт Inter — `font-display: swap`.
- Видео трейлер — не загружать до hover/intersection.
- Кеш SWR/React Query: list — 5 мин, detail — 5 мин, actors/tags popular — 1 ч.
- Prefetch следующей страницы cursor при ≥ 80% скролла.
