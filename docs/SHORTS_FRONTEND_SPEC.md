# ТЗ фронтенда — Shorts (вертикальная лента, «TikTok-style»)

> Дополнение к `docs/FRONTEND_SPEC.md` (API-контракт) и `docs/UI_SPEC.md` (UI/UX). Описывает **отдельный
> раздел** сайта с вертикальными видео и полноэкранным свайп-плеером. Бэкенд реализован полностью —
> ниже актуальный контракт и поведение, реализуется на стороне фронта.

Референс UX: **TikTok / YouTube Shorts / Reels**. Базовые конвенции (язык `?lang`, fallback, action-by-uuid,
абсолютные URL изображений, cursor-пагинация) — те же, что в `FRONTEND_SPEC.md §0`.

---

## 1. Суть фичи

- Появляется класс **вертикальных** видео (флаг `is_vertical=true`). Их проставляет загрузчик при заливке.
- Вертикальные видео **сегрегированы**: они **не попадают** в обычные ленты (`/videos/`, `trending`,
  `popular`, `new`, `recommended`) и в `related`/`next`. Единственный вход в них — раздел **Shorts**.
- Лента Shorts **персонализирована** (как `recommended`) и **не кешируется**.
- **У Shorts НЕТ каталога/сетки.** Нажатие на «Shorts» сразу открывает **первое видео** в полноэкранном
  плеере, дальше — только свайп. Никаких страниц-списков с превью-карточками.
- Shorts можно открыть **в контексте категории/тега**: лента тогда **отфильтрована** по этой категории/тегу
  (см. §2.1, параметры `categories`/`include_tags`), и свайп идёт только внутри неё.

## 2. API-контракт

### 2.1 Лента — `GET /api/v1/videos/shorts/`

| Параметр | Тип | Описание |
|---|---|---|
| `lang` | `ru\|en` | Язык контента (иначе `Accept-Language` → `DEFAULT_LANGUAGE`) |
| `cursor` | str | Курсор следующей/предыдущей страницы (берётся из `next`/`previous`) |
| `page_size` | int | По умолчанию 24, максимум 100 |
| `categories` | slug через запятую | Скоуп ленты по категориям (overlap: совпадает любой) |
| `include_tags` | slug через запятую | Скоуп ленты по тегам (overlap: совпадает любой) |
| `actors` | slug через запятую | (опц.) Скоуп по актёрам |

- **Пагинация — cursor:** `{ "next": "<url|null>", "previous": "<url|null>", "results": [VideoShort] }`.
  Бесконечный свайп = последовательная подгрузка по `next`.
- **Персонализация:** для авторизованного — порядок по истории просмотров (слать JWT `Authorization: Bearer`),
  уже просмотренные исключаются; для анонима — trending-вертикаль. (Алгоритм: теги просмотренных видео →
  ранжирование по глобальному trending; watch%/личные лайки в текущей версии не учитываются.)
- **Скоуп по категории/тегу:** те же slug-фильтры, что и в каталоге (`FRONTEND_SPEC §2.1`). Применяются
  **поверх** персонализации. Вход «Shorts из категории X» → `?categories=<slug-X>`. Без параметров — глобальная
  лента. Фильтры опциональны и комбинируются.
- **Кеш:** отсутствует (как `recommended`). Не полагаться на ETag/повтор — каждый запрос свежий.
- **Auth:** не обязателен (аноним получает trending-вертикаль).

### 2.2 Элемент ленты — `VideoShort`

Это карточка (`VideoCard` из `FRONTEND_SPEC §2.1`) + поле `sources` для инлайн-плеера + флаг `is_vertical`:

```jsonc
{
  "uuid": "ed03078e-…",          // str — для всех действий
  "duration": 42,                 // int (сек)
  "is_vertical": true,            // bool — всегда true в этой ленте
  "views_count": 1200,            // int
  "likes_count": 80,              // int
  "dislikes_count": 5,            // int
  "comments_count": 12,           // int
  "published_at": "2026-06-17T12:00:00Z", // str(iso)|null
  "poster": "https://img.example.com/v/<uuid>/poster.webp", // str|null
  "trailer": "https://v.example.com/v/<uuid>/trailer.mp4",  // str|null
  "title": "…",                   // str (локализованный)
  "slug": "…",                    // str (локализованный) — для шаринга /shorts/<slug>
  "language": "ru",               // str
  "sources": {                    // объект источников плеера
    "mp4": {},                    //   всегда пусто (HLS-only)
    "hls": "https://v.example.com/v/<uuid>/playlist.m3u8", // str|null — мастер-плейлист ABR
    "trailer": "https://v.example.com/v/<uuid>/trailer.mp4" // str|null
  }
}
```

> Отличие от обычной карточки каталога: присутствуют `is_vertical` и **`sources`** (в обычной карточке
> источников нет — там только `poster`+`trailer`). Это позволяет проигрывать сам ролик прямо в ленте,
> без отдельного запроса детали.

### 2.3 Действия (by uuid) — те же, что на детальной странице

Используются существующие user-action эндпоинты (`FRONTEND_SPEC §2.3`), адресация по `uuid`:

| Метод | Путь | Auth | Назначение |
|---|---|---|---|
| POST | `/videos/{uuid}/view/` | — | Засчитать просмотр (дедуп 24ч). Вызывать по порогу залипания, см. §4.5 |
| POST | `/videos/{uuid}/guest-like/` | — | Анонимный лайк (дедуп 7д) |
| POST/DELETE | `/videos/{uuid}/reaction/` | ✔ | Лайк/дизлайк / снять |
| POST/DELETE | `/videos/{uuid}/favorite/` | ✔ | В избранное / убрать |
| POST | `/videos/{uuid}/progress/` | ✔ | Прогресс (throttle ~раз в 10–30с) |
| GET/POST | `/videos/{uuid}/comments/` | —/✔ | Лента/создание комментариев (cursor) |

Состояние лайка/избранного для набора uuid — батч-эндпоинтом состояния (тот же, что для карточек каталога),
чтобы иконки сразу отражали реальность. Счётчики eventually-consistent → **оптимистичные апдейты** на клиенте.

### 2.3.1 Личная библиотека — табы «Видео» / «Shorts»

Вертикальные видео **остаются** в личных лентах пользователя (лайки/избранное/история/продолжить смотреть) —
сегрегация их оттуда не убирает. Чтобы развести их по табам, ленты `/me/*` принимают параметр `?type`:

| Эндпоинт | Параметр | Значения |
|---|---|---|
| `GET /api/v1/me/liked/` | `type` | `videos` (горизонтальные) · `shorts` (вертикальные) · *(опущен)* — оба |
| `GET /api/v1/me/favorites/` | `type` | то же |
| `GET /api/v1/me/history/` | `type` | то же |
| `GET /api/v1/me/continue-watching/` | `type` | то же |

- Таб «Видео» → `?type=videos`, таб «Shorts» → `?type=shorts`. Без параметра — общий список (как раньше,
  обратная совместимость сохранена).
- Ответ — `VideoCard` (PageNumber-пагинация, как у `/me/*`). Для `type=shorts` это карточки **без** `sources`
  (сетка превью библиотеки); открытие в плеер — см. §2.4.
- Неизвестное значение `type` трактуется как «оба».

### 2.4 Детальная страница вертикального видео

`GET /api/v1/videos/{slug}/` для вертикального видео работает как обычно и теперь содержит `is_vertical: true`
(в detail-объекте). Используется для SSR-страницы шаринга `/shorts/<slug>` (canonical/hreflang через `slugs`).
**`related`/`next` для такого видео вернут горизонтальные ролики** (вертикаль из них исключена) — поэтому
навигацию «следующий шорт» строить **из ленты `/videos/shorts/`**, а не через `next`.

## 3. Маршрутизация

| Роут | Назначение | Рендер |
|---|---|---|
| `/shorts` | Глобальная лента: сразу первое видео + свайп | CSR (персонализация, без кеша) |
| `/shorts/<slug>` | Открыть ленту, начав с конкретного ролика (шаринг/диплинк) | SSR для мета/OG, далее CSR-лента |
| `/category/<slug>/shorts` *(или входная кнопка)* | Лента, отфильтрованная по категории (`?categories=<slug>`) | CSR |

- **Никакой страницы-каталога/сетки у Shorts нет.** Любой вход открывает **сразу первый ролик** в
  полноэкранном плеере и дальше работает только свайп.
- Пункт меню **«Shorts»** в шапке (mobile bottom-nav + desktop sidebar/topbar). Иконка — вертикальный
  прямоугольник/«молния». Открывает глобальную ленту.
- **Вход из категории/тега:** на странице категории/тега — кнопка «Shorts», которая открывает ленту
  `GET /videos/shorts/?categories=<slug>` (или `?include_tags=<slug>`); свайп идёт только внутри неё.
  (Маршрут можно сделать вложенным `/category/<slug>/shorts` или модальным оверлеем — на усмотрение фронта.)
- Вертикалки не показываются в обычном каталоге (бэкенд их исключает) → раздел Shorts — единственная точка входа.
- При открытии `/shorts/<slug>`: запросить деталь по slug для первого слайда (или найти его в первой странице
  ленты), затем подгружать `/videos/shorts/` для последующих. Если slug не вертикальный/не найден — редирект на `/shorts`.

## 4. UX плеера (ядро фичи)

### 4.1 Раскладка
- **Полноэкранный вертикальный слайд** 9:16, один ролик = один экран (`height: 100dvh`).
- **Mobile:** видео во весь экран, оверлей-контролы поверх.
- **Desktop:** узкая центральная колонка (макс. ~`min(420px, 56.25vh)` по ширине, 9:16), по бокам затемнённый
  фон; правая колонка действий — рядом со слайдом.

### 4.2 Свайп / навигация
- **Вертикальный свайп:** снизу вверх → следующий ролик, сверху вниз → предыдущий.
- Реализация через **CSS scroll-snap**: контейнер `scroll-snap-type: y mandatory; overflow-y: scroll`,
  каждый слайд `scroll-snap-align: start; height: 100dvh`. Нативно работает с тач-свайпом и колесом мыши.
- **Desktop-управление:** стрелки `↑`/`↓` и колесо — переключение; `Space` — пауза/плей; `M` — мьют;
  `←`/`→` (опц.) — перемотка ±5с.
- Скрывать системный скролл-бар; debounce колеса, чтобы один «щелчок» = один слайд.

### 4.3 Воспроизведение
- Играет **только активный** слайд; соседние на паузе и сброшены к началу (или удержаны в позиции — на выбор).
- **HLS** через тот же механизм, что и основной плеер сайта (Safari — нативно, остальные — hls.js),
  источник — `sources.hls` из элемента ленты.
- **Автоплей в `muted`** (политики браузеров), кнопка «включить звук»; выбор mute/unmute запоминать на сессию
  (`sessionStorage`).
- **Луп** короткого ролика — включён по умолчанию (как в TikTok).
- Тонкий **прогресс-бар** внизу; тап по нему — перемотка (опц.).

### 4.4 Предзагрузка и память
- Держать **смонтированными** текущий ± 1–2 слайда; для остальных — только постер.
- **Прелоад HLS следующего** ролика (manifest + первые сегменты) при приближении к нему.
- Выгружать video-элементы дальних слайдов, чтобы не плодить активные медиа (ограничение по числу `<video>`).
- Дозагрузка страницы по `next` заранее — когда до конца буфера осталось **3–5** роликов.

### 4.5 Засчёт просмотра
- Слать `POST /videos/{uuid}/view/` по **порогу залипания** (например, ролик активен ≥ N секунд или ≥ X%
  длительности) — **не** на каждый промелькнувший слайд. Бэкенд сам дедуплицирует 24ч по IP+UA.
- Прогресс (`/progress/`) — для авторизованных, throttle ~раз в 10–30с активного просмотра.

### 4.6 Жесты (mobile)
- **Тап** — пауза/плей.
- **Двойной тап** — лайк (анимация «сердечка», оптимистично).
- **Долгий тап** — пауза-«peek» (удержание ставит на паузу, отпускание возобновляет).
- **Свайп влево** (опц.) — открыть профиль актёра/тег.

## 5. Оверлей UI (поверх видео)

**Правая вертикальная панель действий** (снизу вверх):
- **Лайк** (+счётчик) — `reaction=like` (auth) / `guest-like` (аноним).
- **Дизлайк** — `reaction=dislike` (auth).
- **Избранное** — `favorite` (auth; аноним → модалка логина).
- **Комментарии** (+счётчик) — открывает нижнюю шторку с лентой комментариев (`/videos/{uuid}/comments/`).
- **Шеринг** — копирует `FRONTEND_BASE_URL/shorts/<slug>` (+ нативный share-sheet на mobile).

**Нижний блок:**
- Заголовок (`title`), актёр(ы)/теги (кликабельны → их страницы), длительность.
- Кнопка **mute/unmute**.

**Состояния:** иконки лайка/избранного отражают реальное состояние пользователя (батч-статус). Для анонима
действия, требующие auth, открывают модалку логина (`FRONTEND_SPEC §1`).

## 6. Граничные случаи
- **Пустая лента** (нет вертикалок / всё просмотрено) — заглушка «Пока нет шортсов» + CTA на обычный каталог.
- **Ошибка HLS** конкретного ролика — авто-скип на следующий, не ломать ленту; залогировать.
- **Конец фида** (`next=null`) — для авторизованного можно повторно запросить (персонализация даст trending-фолбэк);
  показать мягкий индикатор «вы посмотрели всё актуальное».
- **Медленная сеть** — показывать `poster` как плейсхолдер до готовности первого сегмента; спиннер не дольше 1.5с.
- **Поворот экрана / ресайз** — пересчитывать `100dvh` (использовать `dvh`, не `vh`, из-за мобильных адрес-баров).

---

## 7. Производительность / SEO
- `/shorts` — CSR (персонализированный, некешируемый фид). `/shorts/<slug>` — SSR только ради мета/OG-тегов
  первого ролика (canonical, `og:video`, `og:image=poster`), сама лента гидрируется на клиенте.
- Ленивая инициализация hls.js (динамический импорт), один общий пул экземпляров.
- Изображения (`poster`) — `loading="lazy"`, `decoding="async"`.
