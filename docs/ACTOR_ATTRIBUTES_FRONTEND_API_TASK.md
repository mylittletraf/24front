# Фронтенд-задача: новые атрибуты актрис + «общие теги» в API

Хендофф для фронта. Бэкенд расширил **существующие** ответы API (новый публичный эндпоинт **не**
добавлялся). Ничего ломающего: добавлены новые поля и расширено поведение фильтров/тегов.

## 1. Актриса: новые поля

`GET /api/v1/actors/` (список) и `GET /api/v1/actors/<slug>/` (детали) теперь содержат:

| Поле | Тип | Описание |
|---|---|---|
| `ethnicity` | `{uuid,name,slug}` \| `null` | Этничность — атрибут-тег, как `country`/`hair_color`/… |
| `measurements` | `string` | Параметры, свободный текст (напр. `"90-60-90"`). Пусто = `""` |
| `piercings` | `string` | Пирсинг, свободный текст. Пусто = `""` |
| `has_tattoos` | `boolean` | Наличие татуировок |

`ethnicity` сериализуется тем же компактным лейблом, что и прочие атрибуты
(`country`, `body_type`, `bra_size`, `boobs_type`, `hair_color`, `eye_color`), — то есть рендерится
ровно так же, как они.

Пример (фрагмент `GET /api/v1/actors/<slug>/?lang=ru`):
```json
{
  "uuid": "…",
  "name": "Анна",
  "country":   { "uuid": "…", "name": "Россия",  "slug": "russia" },
  "ethnicity": { "uuid": "…", "name": "Азиатка", "slug": "asian" },
  "hair_color": { "uuid": "…", "name": "Блонди", "slug": "blonde" },
  "measurements": "90-60-90",
  "piercings": "пупок",
  "has_tattoos": true
}
```

## 2. Справочник значений атрибутов

`GET /api/v1/actors/attributes/` теперь возвращает дополнительный ключ **`ethnicity`** (список
лейблов тегов), наряду с `country`/`body_type`/`bra_size`/`boobs_type`/`hair_color`/`eye_color`.
Используйте его для выпадающих фильтров.

## 3. Фильтры

- Список актрис: добавлен `?ethnicity=<slug>` (значение — `slug` тега этничности), аналогично
  `?hair_color=<slug>` и т.д.
- Список видео: добавлен `?actor_ethnicity=<slug>` (видео, где у любой актрисы такая этничность),
  аналогично `?actor_hair_color=<slug>` и т.д.

## 4. ГЛАВНОЕ: «общие теги» актрис в тегах видео

У актрисы есть набор **common tags** (контент-теги + категории), которые редактор задаёт **один раз**
на актрисе. Эти теги **автоматически подмешиваются** в теги каждого видео, где она участвует.

Для фронта это **прозрачно**:
- В `GET /api/v1/videos/<slug>/` такие теги приходят в массивах `tags` и `categories` в том же
  формате `{uuid,name,slug}`, что и обычные. **Отличить унаследованный тег от «родного» нельзя и не
  нужно** — спец-обработки не требуется.
- Дубли уже исключены на бэкенде (если тег есть и на видео, и у нескольких актрис — он один).
- Фильтрация работает как с обычными тегами: `GET /api/v1/videos/?include_tags=<slug>`,
  `?exclude_tags=<slug>`, `?categories=<slug>` находят/исключают и видео, где тег пришёл от актрисы.
- `studios` и атрибуты актрис в общие теги не попадают (туда кладут только контент-теги и категории).

Никаких действий на фронте по этому пункту не требуется — поведение «само заработает» на
существующей вёрстке тегов/категорий и фильтрах. (Собственный список common-тегов на странице
актрисы публичный API отдельно сейчас не отдаёт — теги видны там, где и должны: в тегах её видео.)

## 5. Чек-лист приёмки
1. Карточка/страница актрисы показывает `ethnicity`, `measurements`, `piercings`, `has_tattoos`.
2. Фильтр актрис по этничности и фильтр видео `actor_ethnicity` работают.
3. На странице вид, где у актрисы заданы common-теги, эти теги/категории видны в общем списке без
   дублей, и по ним фильтруется каталог.
