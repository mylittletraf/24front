# Бэкенд-задача: page-number пагинация для каталога видео (нумерованные страницы)

Хендофф для бэкенда. На фронте делаем **классическую нумерованную пагинацию** (1, 2, 3 … N) для
листингов видео (главная, страницы тега/категории/студии). Сейчас `/videos/` отдаёт **только
cursor**-пагинацию (`next`/`previous`, без `count` и без номеров страниц) — `?page=`/`?offset=`
игнорируются (проверено curl). Нумерованная пагинация требует знать **общее число** и уметь прыгать
на страницу N — это нужно добавить на бэке.

## Что нужно
Поддержать **page-number режим** на `GET /videos/`:
- Принимать `?page=N` (1-based) и существующий `?page_size=` (как сейчас).
- В этом режиме отвечать DRF-формой PageNumberPagination:
  ```json
  { "count": 1234, "next": "<url|null>", "previous": "<url|null>", "results": [ … ] }
  ```
  `count` — общее число подходящих видео (для расчёта числа страниц).
- Работает со ВСЕМИ текущими фильтрами и сортировкой: `include_tags`, `exclude_tags`, `categories`,
  `actors`, `actor_*` атрибуты, `q`, `duration_*`, `published_*`, `sort`, `lang`.
- Детерминированный порядок при пагинации по страницам (стабильная сортировка, чтобы страница N не
  «плыла» между запросами в пределах одной выборки).

## Важно про совместимость (cursor оставить по умолчанию)
`/videos/` сейчас курсорный, и на cursor завязаны другие потребители (бесконечная лента шортсов,
related, ленты `/me/*` и т.п.). **Не ломать их.** Варианты переключения режима — на выбор бэка,
сообщите какой:
1. Режим включается **наличием `?page=`** (есть `page` → PageNumber; иначе cursor по-умолчанию). ← удобно фронту.
2. Либо явный флаг `?pagination=page` (или `?paginate=page`).
3. Либо отдельный путь/параметр — но тогда нужно, чтобы он принимал те же фильтры.

Достаточно покрыть **`/videos/`** — фронт строит и главную, и страницы тега/категории/студии через
`/videos/` с фильтрами (отдельные `/{entity}/{slug}/videos/` фронт не использует для листинга).

## Критерии приёмки
- `GET /videos/?page=1&page_size=24` → `{count, next, previous, results}` с `count` = всего видео,
  `results` = первые 24.
- `GET /videos/?page=2&page_size=24&categories=molodaia&sort=newest` → второй блок из 24 по этому
  фильтру; `count` = всего видео в фильтре; порядок стабильный (page 1 и page 2 не пересекаются).
- Существующий **курсорный** ответ без `?page` не меняется (шортсы/related/`/me` работают как раньше).
- Сообщите фронту триггер режима (п.1/п.2/п.3) и, если поменялись имена/поведение, — детали.

## Что делает фронт после этого (для контекста, не задача бэка)
- Серверный рендер текущей страницы + панель номеров «1 2 3 … N».
- `self-canonical` на каждой странице (`?page=2` канонично само на себя), вероятно `noindex,follow`
  на страницах ≥2 (бот ходит по ссылкам, листинги не индексируются — SEO-контент на `/video/`,
  `/tag/`, `/category/`). Это снимает текущую «кривую» связку «Загрузить ещё» + «Старше/Новее».
